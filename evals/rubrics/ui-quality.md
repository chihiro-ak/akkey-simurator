# ui-quality rubric

Review each item as PASS / FAIL / PARTIAL.

## Layout
- No clipped text or controls.
- No accidental overflow.
- Spacing is consistent with existing UI.
- Visual hierarchy is clear.

## Interaction
- Primary CTA is obvious.
- Disabled and loading states communicate clearly.
- Error messages are actionable.

## Consistency
- Existing design tokens are reused.
- Existing component patterns are reused where possible.
