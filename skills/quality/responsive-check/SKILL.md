# responsive-check

## Purpose
Catch layout and interaction issues caused by viewport changes before the feature is closed.

## Inputs
- Running UI or screenshots
- Changed files
- Relevant feature spec

## Outputs
- Viewport-by-viewport findings
- PASS / FAIL / PARTIAL result
- Recommended fixes

## Required viewports
- 360px width
- 390px width
- 768px width
- Desktop

## Required checks
1. No accidental horizontal overflow.
2. No clipped text, icons, or buttons.
3. Form fields remain usable and readable.
4. Primary CTA remains visible and understandable.
5. Sticky headers, drawers, and dialogs do not cover required content.
6. Spacing and hierarchy remain coherent.
