# 01_product_concept.md

## 概要
アクリルキーホルダーを制作する個人クリエイター向けに、画像アップロード、金具選択、穴位置調整を行い、完成イメージと揺れ方の傾向をブラウザ上で確認できるシミュレーターを提供する。

MVP では「専門的な 3D ツールを使わずに、入稿前の不安を短時間で減らせること」を最重要価値とする。高精度な物理再現ではなく、見た目確認と位置検討に十分な軽量プレビュー体験を優先する。

## 解決したい課題
- 穴位置によって見た目や揺れ方がどう変わるかを制作前に判断しづらい
- 完成イメージを事前確認する手段が限られ、試作コストが発生しやすい
- Blender などの 3D ツールは学習コストが高く、個人クリエイターには重い

## ターゲットユーザー
### Primary
- 同人グッズを作成するイラストレーター
- ライバー、配信者、イベント主催者など、短期間でグッズ案を確認したい個人制作者
- 入稿前に完成イメージを確認したい個人クリエイター

### Secondary
- 複数案を比較して制作支援に活かしたい小規模ショップ運営者
- 将来的に制作相談や有料支援機能を使う可能性があるユーザー

## ユーザー価値仮説
- 画像をアップロードしてすぐに見た目を確認できれば、制作前の不安が減る
- 金具の種類と穴位置を少ない操作で試せれば、試作前の比較検討がしやすくなる
- 簡易な揺れ表現でも、静止画だけでは分からない違和感に気づける

## 主要ユースケース
1. ユーザーがイラスト画像をアップロードする
2. キーホルダーに使う金具を選ぶ
3. 本体と金具の接続位置を調整する
4. 完成イメージをプレビューし、動きの傾向を確認する
5. 必要ならプレビュー結果を再調整する

## MVP の体験定義
### 提供するもの
- 単体アクリルキーホルダーのシミュレーション
- 画像アップロード
- 金具選択
- 穴位置調整
- 完成イメージのプレビュー
- 簡易な揺れアニメーションまたは揺れ確認 UI

### 提供しないもの
- 高精度な物理シミュレーション
- 入稿データの完全自動生成
- 認証、保存、課金
- 2連アクキーの対応

## 成功指標案
- 初見ユーザーが迷わず 5 分以内に 1 回シミュレーションを完了できる
- 2 か月以内に 100 人へ体験提供できる
- MVP 利用後に「穴位置や見た目の検討に役立った」と判断できる定性フィードバックを収集できる

## UX 方針
- 1 画面または少数画面で主要導線を完了できる構成を優先する
- 操作順序を明示し、アップロード、選択、配置、確認のステップを分かりやすく見せる
- ドラッグ操作に加えて、失敗しにくい補助 UI を用意する前提で設計する
- 通常、empty、error、success の各状態が分かる UI を前提にする
- モバイル幅でも主要操作が成立するレイアウトにする

## 想定 IA
- シミュレーション作成画面
  - 画像アップロード
  - 金具選択
  - 穴位置調整
  - シミュレーション実行
- プレビュー画面
  - 3D 風プレビューまたは簡易立体プレビュー
  - 戻る
  - 公開または埋め込み用の導線プレースホルダ

## 将来拡張の方向性
- 2連アクキー対応
- より自然な揺れや回転を含む物理表現
- 結果の共有 URL、埋め込み、保存
- 制作支援、有料オプション、入稿補助

## 前提と制約
- 技術スタックは Next.js / React / TypeScript / Tailwind CSS / shadcn/ui
- 個人開発前提のため、初期は実装と検証コストを抑える
- UI を含む feature は実装前に mock 承認が必要

## 現時点の不足情報
- 対応する画像形式、容量制限、透過前提の有無
- 金具種類の初期候補数と、各金具の見た目差分の扱い
- 穴位置の制御方法
  - ドラッグのみか
  - 数値入力やスナップ補助を持たせるか
- プレビューの実装レベル
  - 2D ベース疑似 3D か
  - 軽量 3D 表示か
- 「公開する」の具体仕様
  - URL 共有か
  - 埋め込みコード生成か
  - 将来導線の仮ボタンか

## 企画判断メモ
- MVP では「制作前の検討支援」を成立させることを優先し、共有機能は backlog 上の拡張候補として扱う
- 2連アクキーはニーズがあるが、穴位置調整とプレビューの難易度を上げるため初回リリースでは対象外とする
- UI イメージ上の「公開する」は価値仮説として残しつつ、MVP の必須要件には含めない
