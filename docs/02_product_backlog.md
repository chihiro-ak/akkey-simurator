# 02_product_backlog.md

## 方針
`docs/00_product.md` と UI イメージをもとに、MVP の主要導線を成立させるための backlog を整理する。UI を含む feature は実装前に `docs/features/*-ui-mock.md` の作成と承認を前提とする。

## エピック一覧
- EPIC-01: シミュレーション作成導線の確立
- EPIC-02: プレビュー体験の成立
- EPIC-03: 品質確認とリリース準備
- EPIC-04: MVP 後の拡張準備

## MVP feature 一覧
- FEAT-001: 画像アップロードとシミュレーション初期設定
- FEAT-002: 金具選択と穴位置調整
- FEAT-003: 完成イメージのプレビュー

## Backlog
### BL-001
- 種別: product
- タイトル: MVP のシミュレーション対象と成功条件を確定する
- エピック: EPIC-01
- 優先度: High
- 依存関係: なし
- 概要: 単体アクキーに限定した MVP の対象範囲、操作完了条件、共有機能の扱いを確定する
- DoD:
  - 単体アクキーが MVP 対象であると docs に明記されている
  - 2連アクキーが out of scope として整理されている
  - 「公開する」の扱いが MVP 必須か将来導線か明記されている
- 備考: `01_product_concept` と feature spec の基準になる

### BL-002
- 種別: design
- タイトル: シミュレーション作成画面の UI mock を作成する
- エピック: EPIC-01
- 優先度: High
- 依存関係: BL-001
- 概要: アップロード、金具選択、穴位置調整、実行 CTA を含む画面 mock を作成する
- DoD:
  - 通常、empty、error、success の状態が整理されている
  - モバイル時の構成が整理されている
  - ドラッグ操作と補助 UI の想定が記載されている
  - 実装開始可否が判断できる
- 備考: 実装前必須

### BL-003
- 種別: implementation
- タイトル: 画像アップロードを実装する
- エピック: EPIC-01
- 優先度: High
- 依存関係: BL-002
- 概要: 対応形式の画像を読み込み、シミュレーションの元画像として表示できるようにする
- DoD:
  - 画像未選択時の empty state がある
  - 読み込み成功時にプレビュー用画像が表示される
  - 非対応または読込失敗時の error state がある
  - モバイル幅でも操作できる
- 備考: 透過 PNG 前提などの詳細は別途確定が必要

### BL-004
- 種別: implementation
- タイトル: 金具選択 UI を実装する
- エピック: EPIC-01
- 優先度: High
- 依存関係: BL-002
- 概要: 初期候補の金具を選択し、見た目に反映できるようにする
- DoD:
  - 少なくとも MVP で定義した金具候補を選択できる
  - 選択中状態が視覚的に判別できる
  - 金具未選択時の扱いが定義されている
  - 画像との組み合わせ確認ができる
- 備考: 金具マスタの持ち方は実装時検討

### BL-005
- 種別: implementation
- タイトル: 穴位置調整 UI を実装する
- エピック: EPIC-01
- 優先度: High
- 依存関係: BL-002, BL-003, BL-004
- 概要: 本体画像に対して金具接続位置を調整し、プレビュー反映できるようにする
- DoD:
  - ユーザーが接続位置を変更できる
  - 位置変更結果が即時に画面へ反映される
  - 画面外や不正位置になりにくい制御がある
  - 調整不能時の補助メッセージがある
- 備考: ドラッグのみだと操作難度が高い可能性がある

### BL-006
- 種別: design
- タイトル: プレビュー画面の UI mock を作成する
- エピック: EPIC-02
- 優先度: High
- 依存関係: BL-001
- 概要: 完成イメージ確認、戻る導線、将来の共有導線を含むプレビュー画面 mock を作成する
- DoD:
  - 通常、loading、error の状態が整理されている
  - プレビュー対象と操作可能領域が分かる
  - モバイル時の情報優先度が整理されている
  - 戻る動線と再調整動線が整理されている
- 備考: 実装前必須

### BL-007
- 種別: implementation
- タイトル: 完成イメージの簡易プレビューを実装する
- エピック: EPIC-02
- 優先度: High
- 依存関係: BL-006, BL-003, BL-004, BL-005
- 概要: 画像、金具、穴位置をもとに、完成イメージを一体で確認できるようにする
- DoD:
  - シミュレーション結果が一画面で確認できる
  - 戻る操作で設定画面に戻れる
  - 設定変更が再度プレビューに反映される
  - 必要データ不足時のエラー表示がある
- 備考: 初回は軽量な 2D/疑似 3D でも可

### BL-008
- 種別: implementation
- タイトル: 簡易な揺れ確認表現を実装する
- エピック: EPIC-02
- 優先度: Medium
- 依存関係: BL-007
- 概要: 穴位置による印象差を掴める程度の簡易アニメーションまたは揺れ確認 UI を実装する
- DoD:
  - 少なくとも 1 パターンの揺れ確認ができる
  - 高負荷になりすぎない
  - 動きが見えない場合の代替表現がある
- 備考: 高精度物理は対象外

### BL-009
- 種別: docs
- タイトル: MVP feature spec を整備する
- エピック: EPIC-01
- 優先度: High
- 依存関係: BL-001
- 概要: feature ごとの scope、out of scope、done criteria を docs/features に整理する
- DoD:
  - 各 feature に固有 ID がある
  - 実装判断に必要な scope が明示されている
  - UI mock 前提が明記されている
- 備考: 今回の成果物に含む

### BL-010
- 種別: qa
- タイトル: シミュレーション主要導線の確認観点を作成する
- エピック: EPIC-03
- 優先度: Medium
- 依存関係: BL-003, BL-004, BL-005, BL-007
- 概要: empty、error、success、モバイル確認を含む QA 観点を定義する
- DoD:
  - 各 feature の主要状態を確認できる
  - モバイル確認項目がある
  - overflow と CTA の確認項目がある
- 備考: 実装後に詳細化

### BL-011
- 種別: test
- タイトル: build、typecheck、主要表示確認の検証フローを整備する
- エピック: EPIC-03
- 優先度: Medium
- 依存関係: BL-003, BL-004, BL-005, BL-007
- 概要: 実装完了条件として必要な静的検証と主要導線確認のフローを揃える
- DoD:
  - build と typecheck の実行方法が整理されている
  - 主要導線の確認ポイントがある
  - 再発防止観点のテスト追加判断基準がある
- 備考: `00_dev_rules.md` 準拠

### BL-012
- 種別: product
- タイトル: 2連アクキーの将来拡張要件を整理する
- エピック: EPIC-04
- 優先度: Low
- 依存関係: BL-007
- 概要: 2連構成に必要なデータ構造、UI、プレビュー仕様の差分を整理する
- DoD:
  - MVP との差分が整理されている
  - UI と技術の難所が明記されている
  - 次期検討の入口になる
- 備考: 初回リリース対象外

### BL-013
- 種別: product
- タイトル: 共有、公開、埋め込み機能の要件を整理する
- エピック: EPIC-04
- 優先度: Low
- 依存関係: BL-007
- 概要: UI イメージにある公開導線を、次フェーズで具体要件へ落とし込む
- DoD:
  - URL 共有、埋め込み、画像出力などの選択肢が比較されている
  - 認証不要で成立する最小案が整理されている
  - セキュリティと運用上の前提が整理されている
- 備考: MVP では仮ボタンまたは非表示でも可

## 依存関係メモ
- 実装系 backlog は対応する UI mock 完了後に着手する
- 穴位置調整は画像アップロードと金具選択が前提
- プレビューは設定入力系 feature 完了後に着手する

## 不足情報
- 対応画像形式と容量上限
- 金具候補の具体種類
- 穴位置調整の制約ルール
- 3D 表現レベルの許容ライン
- 公開機能の扱い

## リスク
- 穴位置調整 UI が直感的でないと、主要価値が落ちる
- 物理表現を追いすぎると MVP の実装コストが急増する
- 画像前提や透過処理の仕様未確定が、UI と実装を手戻りさせる

## 次アクション
- FEAT-001 から FEAT-003 の UI mock を作成する
- 画像仕様と金具候補の最小セットを決める
- プレビューを 2D ベース疑似 3D にするか軽量 3D にするかを決める
