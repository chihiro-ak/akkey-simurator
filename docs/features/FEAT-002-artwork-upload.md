# FEAT-002 artwork upload

## タイトル
ユーザーが画像をアップロードして失敗なく読み込める

## 背景
- MVP の最初の入口は画像アップロードであり、ここでつまずくと後続価値に到達できない
- 現状 runnable UI は画像選択自体は可能だが、対応形式、失敗時表示、再選択導線が未整理である

## 目的
- ユーザーが自分のイラスト画像を安全に読み込み、プレビューの起点にできるようにする

## scope
- `png`, `jpg/jpeg`, `webp` の受け付け
- アップロード前の empty UI
- アップロード成功後のプレビュー反映
- 非対応形式や読み込み失敗時のエラーメッセージ
- 画像差し替え導線
- 背景透過推奨など最低限のガイド文言

## out of scope
- 画像トリミング
- 背景除去
- サーバー保存
- 複数画像管理

## done criteria
- 対応形式の画像を選択すると画面上のプレビューに反映される
- 非対応形式や読み込み失敗時に日本語で理由が分かる
- 画像未選択時と選択済み時で UI の見え方が変わる
- 差し替え操作が明示されている
- モバイルでもファイル入力が使いにくくならない

## 検証方法
- `npm run dev` でアプリを起動し、対応形式の画像アップロード、差し替え、エラー時表示を手動確認する
- Playwright E2E で画像選択後にプレビューへ反映されること、未選択時と選択済み時で表示が変わることを確認する
- 非対応形式や読み込み失敗を再現できる場合は、そのエラー表示を確認する
- 他 feature の基本導線として、パーツ選択とプレビュー切替が継続して利用できることを確認する

## feature completion criteria
- 単体の検証が上記方法で完了している
- `npm run dev` でアプリが立ち上がる
- 既存導線に影響がないことを確認できている
- ユーザーの手元でレビュー、動作確認が完了している

## UI states
- empty: 画像未選択、推奨形式を表示
- success: 画像名または選択済み状態を表示
- error: 非対応形式、破損ファイル、読み込み失敗
- disabled: 後続操作が未入力のため限定される場合の表示
