# FEAT-005 preview and motion

## タイトル
ユーザーが完成イメージと揺れ方の傾向を確認できる

## 背景
- ユーザー価値は「完成後の見た目」と「揺れ方の傾向」が事前に分かることにある
- MVP では高精度物理ではなく、判断材料になる軽量なプレビュー体験を優先する

## 目的
- 画像、金具、穴位置の結果を統合し、静止状態と簡易な動き確認の両方を提供する

## scope
- 編集モードとプレビューモードの切り替え
- 完成イメージの静止プレビュー
- 穴位置に応じて見え方が変わる簡易な揺れアニメーション
- プレビュー補助文言
- 入力不足時の案内表示

## out of scope
- 高精度物理演算
- 実時間の重力・衝突計算
- 動画書き出し
- 共有 URL の生成

## done criteria
- 必須入力が揃うと完成イメージを確認できる
- 編集モードとプレビューモードの違いが分かる
- 穴位置や金具の変更がプレビューに反映される
- 揺れの表現が「傾向確認」として理解できる補助説明を伴う
- empty / error / ready の各状態で表示破綻がない

## 検証方法
- `npm run dev` でアプリを起動し、編集モードとプレビューモードの切り替え、揺れ確認、入力不足時表示を手動確認する
- Playwright E2E で画像アップロード、金具選択、穴位置調整の後にプレビュー結果へ到達できることを確認する
- empty / error / ready の状態別に表示破綻がないことを確認する
- 既存導線として画像アップロードと金具選択の動作が維持されていることを確認する

## feature completion criteria
- 単体の検証が上記方法で完了している
- `npm run dev` でアプリが立ち上がる
- 既存導線に影響がないことを確認できている
- ユーザーの手元でレビュー、動作確認が完了している

## UI states
- empty: 画像や位置が未設定で、必要な入力を案内する
- ready: 静止プレビューと揺れ確認ができる
- error: 入力不整合でプレビューできない理由が分かる
- success: ユーザーが再調整可能な状態で結果を確認できる
