# FEAT-004 hole position adjustment

## タイトル
ユーザーが穴位置を調整して接続位置を検討できる

## 背景
- 本プロダクトの核は「どこに穴をあけるとどう見えるか」の検討支援にある
- 現状 runnable UI にはドラッグ文言だけあり、実際の接続位置調整は未実装である

## 目的
- ユーザーが本体画像に対する接続位置を調整し、見た目の違いをその場で確認できるようにする

## scope
- 穴位置を示すハンドルまたは接続ポイント UI
- ドラッグによる位置調整
- 編集モード上での本体画像ドラッグによる表示位置調整
- 補助的な数値表示または位置ガイド
- 調整結果のプレビュー即時反映
- 極端な位置を避けるための制御

## out of scope
- 複数穴の管理
- 自動最適配置
- 画像輪郭からの自動当たり判定
- 2連アクキー用の複数接続

## done criteria
- ユーザーが穴位置を変更するとプレビューに即時反映される
- ユーザーが編集モードで本体画像をドラッグすると表示位置を調整できる
- モバイルでも致命的に操作しづらくない
- 誤操作しやすい場合に補助表示がある
- 有効範囲外へは移動できない、または安全に補正される
- 位置変更後に FEAT-005 の揺れ確認と整合する

## 検証方法
- `npm run dev` でアプリを起動し、マウス操作とタッチ相当の狭い幅で穴位置調整を手動確認する
- `npm run dev` で編集モードの画像ドラッグ操作がマウス / タッチの両方で成立することを確認する
- Playwright E2E で位置調整操作後にプレビューが更新されること、未入力時は無効化または案内が出ることを確認する
- 有効範囲外への移動時に破綻しないこと、補正または制限が働くことを確認する
- 画像アップロード、金具選択の既存導線が壊れていないことをスモーク確認する

## feature completion criteria
- 単体の検証が上記方法で完了している
- `npm run dev` でアプリが立ち上がる
- 既存導線に影響がないことを確認できている
- ユーザーの手元でレビュー、動作確認が完了している

## UI states
- empty: 画像未選択時は調整 UI を無効化する
- editing: 位置調整中で数値やガイドが見える
- success: 最終位置が保持されている
- error: 画像未選択や位置不整合時に案内を出す

## 依存
- FEAT-001
- FEAT-002

## メモ
- 数値入力とスナップ補助は不足情報だが、MVP ではドラッグ + 最低限のガイドから始めるのが妥当
- 穴位置調整用ハンドルと本体画像ドラッグの責務が混線しないよう、操作対象の見分けを UI 上で明確にする
