# FEAT-003 hardware selection

## タイトル
ユーザーが金具を選んで見た目の違いを比較できる

## 背景
- 金具の種類は完成イメージの印象に直結し、ユーザーの比較ニーズが高い
- runnable UI は複数候補からの選択 UI を前提にしている

## 目的
- ユーザーが候補金具を切り替えながら、見た目差分をすぐ比較できるようにする

## scope
- `ナスカン`, `ボールチェーン`, `ストラップ` の 3 種を候補として表示する
- `parts/` 配下の各画像を利用し、一覧では見分けやすいようにトリミングして表示する
- 選択状態の明示
- 編集画面 / プレビュー画面の両方への選択金具の表示反映
- 候補ごとのラベルと視覚差分
- 未選択時の初期値またはガイド表示

## out of scope
- 金具カスタム追加
- 色や素材の詳細指定
- 2連アクキー用の接続パーツ管理
- パーツ画像自体のドラッグ移動

## done criteria
- ユーザーが候補を切り替えると選択状態が明確に変わる
- `ナスカン`, `ボールチェーン`, `ストラップ` の 3 種のみが候補として表示される
- 候補一覧で各パーツ画像が大きすぎず、トリミング後も種別が判別できる
- 編集画面とプレビュー画面の両方で選択中パーツが即時更新される
- 候補数がモバイル幅で見切れず操作できる
- 初見で「今どの金具を見ているか」が分かる
- FEAT-005 のプレビューと状態整合が取れている
- パーツ表示の位置変更のためのドラッグ操作を要求しない

## 検証方法
- `npm run dev` でアプリを起動し、各金具候補を切り替えて選択状態とプレビュー反映を目視確認する
- `npm run dev` で `parts/ナスカン.png`, `parts/ボールチェーン.png`, `parts/ストラップ.png` が一覧とプレビューに反映されることを確認する
- `npm run dev` で候補一覧のトリミング後も各パーツ種別を識別できることを確認する
- Playwright E2E で候補の切り替え、選択中スタイル、プレビュー反映の基本動線を確認する
- モバイル幅で候補が見切れず、タップ操作できることを確認する
- 画像アップロード済み状態と未アップロード状態の両方で表示破綻がないことを確認する

## feature completion criteria
- 単体の検証が上記方法で完了している
- `npm run dev` でアプリが立ち上がる
- 既存導線に影響がないことを確認できている
- ユーザーの手元でレビュー、動作確認が完了している

## UI states
- empty: 初期候補が自動選択されるか、選択を促す表示がある
- success: 選択中の金具名が分かる
- error: 候補データ取得失敗や設定不整合時のフォールバック表示

## 依存
- FEAT-001

## メモ
- パーツ素材は `parts/` 配下の実画像を正とし、UI 表示用に必要最小限のトリミングで整える
- プレビュー上のパーツは選択結果の表示のみを担当し、位置変更のためのドラッグ対象にはしない
